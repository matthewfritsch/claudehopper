<task type="auto" tdd="true">
  <name>Task 1: Create internal/usage package with RecordUsage and ReadUsage</name>
  <files>internal/usage/usage.go, internal/usage/usage_test.go</files>
  <behavior>
    - RecordUsage appends a valid JSON line to configDir/usage.jsonl with profile, timestamp (RFC3339Nano), and action fields
    - RecordUsage creates configDir via os.MkdirAll if missing (first-run scenario)
    - RecordUsage never panics or returns error on write failure (e.g., read-only fs simulated by invalid path)
    - ReadUsage reads all lines from usage.jsonl and returns []UsageEntry
    - ReadUsage returns empty slice (not error) when file does not exist
    - ReadUsage skips malformed lines silently
  </behavior>
  <action>
    Create `internal/usage/` directory with `usage.go` containing:

    1. `UsageEntry` struct: `Profile string`, `Timestamp string`, `Action string` with json tags (`profile`, `timestamp`, `action`).

    2. `RecordUsage(configDir, profileName, action string)` — void return, swallows all errors:
       - `os.MkdirAll(configDir, 0755)` first
       - Create entry with `time.Now().Format(time.RFC3339Nano)` timestamp
       - `json.Marshal` the entry
       - Open `filepath.Join(configDir, "usage.jsonl")` with `os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644`
       - Write marshaled bytes + newline
       - Swallow ALL errors at every step — never panic, never return error

    3. `ReadUsage(configDir string) ([]UsageEntry, error)`:
       - Read `usage.jsonl` from configDir
       - If file does not exist, return empty slice, nil error
       - Parse each line as UsageEntry via json.Unmarshal
       - Skip lines that fail to parse (malformed)
       - Return parsed entries

    Write tests in `internal/usage/usage_test.go` using t.TempDir() for isolation:
    - TestRecordUsage: record one entry, read file back, verify JSON structure
    - TestRecordUsage_CreatesDir: pass non-existent subdir, verify it creates it
    - TestRecordUsage_NoError: pass "/dev/null/impossible" as configDir, verify no panic
    - TestRecordUsage_MultipleAppends: record 3 entries, verify 3 lines in file
    - TestReadUsage_FileNotExist: returns empty slice, nil error
    - TestReadUsage_SkipsMalformed: file with valid + invalid lines, returns only valid
  </action>
  <verify>
    <automated>cd /home/matthew/Programming/claudehopper-go && go test ./internal/usage/ -v -timeout 30s</automated>
  </verify>
  <done>internal/usage package exists with RecordUsage (void, error-swallowing) and ReadUsage (returns entries). All 6+ tests pass.</done>
</task>
