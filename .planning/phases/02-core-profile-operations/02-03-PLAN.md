  <name>Task 1: Preflight validation, backup, and link managed path</name>
  <files>internal/profile/switch.go, internal/profile/switch_test.go</files>
  <behavior>
    - Test: ValidatePreflight returns nil when all managed paths exist in target profile dir
    - Test: ValidatePreflight returns error listing missing paths when some managed paths are absent
    - Test: backupPath returns "file.hop-backup" when no backup exists
    - Test: backupPath returns "file.hop-backup.1" when "file.hop-backup" already exists
    - Test: backupPath returns "file.hop-backup.2" when both .hop-backup and .hop-backup.1 exist
    - Test: backupPath uses os.Lstat (detects dangling symlinks as existing)
    - Test: linkManagedPath creates symlink from claudeDir/name -> absolute path of profileDir/name for regular files
    - Test: linkManagedPath preserves symlink target (os.Readlink) for symlinks in profile dir
    - Test: linkManagedPath backs up conflicting real file before creating symlink
    - Test: linkManagedPath backs up conflicting real directory before creating symlink
    - Test: DetectUnmanaged returns files in claudeDir that are not in managed_paths, not protected, not .hop-*, not .ccswap*, not ending in .hop-backup*
    - Test: DetectUnmanaged skips symlinks pointing into shared dir
  </behavior>
  <action>
1. In `internal/profile/switch.go`, implement:

   - `type SwitchAction struct { Action, Path, Detail string }` where Action is "link", "unlink", "backup", "orphan", "skip"

   - `func ValidatePreflight(profileDir string, manifest config.Manifest) ([]SwitchAction, error)`:
     - For each path in manifest.ManagedPaths, check os.Lstat(filepath.Join(profileDir, path))
     - If missing: collect as error. If all present: return list of planned actions
     - Also check claudeDir state for each path to determine if backup/unlink/link needed
     - Actually needs claudeDir too. Signature: `func ValidatePreflight(profileDir, claudeDir string, manifest config.Manifest) ([]SwitchAction, error)`

   - `func backupPath(path string) string`:
     - Try `path + ".hop-backup"`, then `.hop-backup.1`, `.hop-backup.2`, etc.
     - Use os.Lstat (not os.Stat) to detect dangling symlinks

   - `func linkManagedPath(profileDir, claudeDir, name string) (backedUp bool, err error)`:
     - Check what exists at claudeDir/name via os.Lstat
     - If real file or real dir: backup via os.Rename to backupPath, set backedUp=true
     - If symlink (to wrong target): os.Remove it
     - Determine target: if profileDir/name is a symlink, use os.Readlink(profileDir/name); else use filepath.Abs(profileDir/name)
     - Create symlink via fs.AtomicSymlink(target, claudeDir/name)

   - `func DetectUnmanaged(claudeDir, sharedDir string, managedPaths []string) ([]string, error)`:
     - Read claudeDir entries
     - Filter out: entries in managedPaths set, fs.IsProtected(name), strings.HasPrefix(name, ".hop-"), strings.HasPrefix(name, ".ccswap"), strings matching `.hop-backup` pattern (contains ".hop-backup")
     - Filter out: symlinks whose target (os.Readlink) starts with sharedDir path
     - Return sorted list of unmanaged file names

2. Write tests in `internal/profile/switch_test.go` using t.TempDir() for all behavior cases above. Create actual file/dir/symlink structures to test against.

  <behavior>

    - Test: DoSwitch with same profile and Force=false returns "already active" error
    - Test: DoSwitch with same profile and Force=true re-links all paths
    - Test: DoSwitch unlinks current profile's managed paths, then links target profile's managed paths
    - Test: DoSwitch saves new active profile to config
    - Test: DoSwitch with conflicting files creates .hop-backup before linking
    - Test: DoSwitch returns SwitchResult with summary of actions taken
    - Test: AdoptUnmanaged adds files to departing profile's manifest managed_paths
  </behavior>
  <action>
1. Add to `internal/profile/switch.go`:

   - `type SwitchOptions struct { DryRun, Force bool; AdoptFiles []string }` - AdoptFiles is populated by the CLI layer after prompting the user
   - `type SwitchResult struct { Actions []SwitchAction; BackedUp []string; Adopted []string }`

   - `func DoSwitch(profilesDir, claudeDir, configPath, sharedDir, targetName, currentName string, opts SwitchOptions) (*SwitchResult, error)`:
     Full sequence matching Python `_do_switch`:
     1. If currentName == targetName && !opts.Force: return error "Already on 'name'. Use --force to re-link."
     2. Load target manifest from profilesDir/targetName/.hop-manifest.json
     3. Call ValidatePreflight(targetProfileDir, claudeDir, targetManifest) -- if errors, return them
     4. If opts.DryRun: return result with planned actions, no writes
     5. If currentName != "" (has current profile):
        a. Call DetectUnmanaged for informational purposes (caller handles prompt)
        b. If opts.AdoptFiles is non-empty, call AdoptUnmanaged to add them to current profile's manifest
        c. Unlink current profile's managed paths: for each managed path in current manifest, os.Remove(claudeDir/path) -- only remove symlinks, skip real files
     6. For each path in target manifest ManagedPaths: call linkManagedPath
     7. SaveConfig with Active=targetName
     8. Return SwitchResult with summary

   - `func AdoptUnmanaged(profileDir string, manifest *config.Manifest, files []string) error`:
     - For each file in files: add to manifest.ManagedPaths
     - Save manifest
     - Move files from claudeDir to profileDir (they become managed)
     Wait -- adopt means "add to the departing profile's manifest so they're tracked." The files are already in claudeDir. We need to move/copy them into the departing profile dir AND add to its managed_paths. Then on the next switch back, they'll be linked.
     - For each file: os.Rename(claudeDir/file, profileDir/file), append to manifest.ManagedPaths
     - SaveManifest

2. Add to `internal/profile/switch_test.go`:

   - DryRun test: verify no filesystem changes after call
   - Force test: same profile re-links
   - Backup test: place real file at conflict location, verify .hop-backup created
   - AdoptUnmanaged test: verify files moved and manifest updated
