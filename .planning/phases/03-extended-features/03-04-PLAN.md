  <name>Task 1: Implement AggregateStats in internal/usage and UnmanageActive in internal/profile</name>
  <files>internal/usage/usage.go, internal/usage/usage_test.go, internal/profile/unmanage.go, internal/profile/unmanage_test.go</files>
  <behavior>
    - AggregateStats reads usage.jsonl and returns per-profile switch counts and last-used timestamps
    - AggregateStats --since filter excludes entries with timestamps before the specified date
    - AggregateStats --profile filter includes only entries for the specified profile name
    - AggregateStats returns results sorted by switch count descending
    - UnmanageActive materializes all symlinks in claudeDir to real files (follows symlinks to real content)
    - UnmanageActive skips non-symlinks in claudeDir (they are already real files)
    - UnmanageActive handles both file and directory symlink targets
    - UnmanageActive sets active to empty string in config.json
    - UnmanageActive dry-run lists what would be materialized without touching filesystem
  </behavior>
  <action>
    **Add to internal/usage/usage.go:**

    1. Define result types:
       ```go
       type ProfileStats struct {
           Name     string            `json:"name"`
           Switches int               `json:"switches"`
           LastUsed string            `json:"last_used"`
           Actions  map[string]int    `json:"actions"`
       }

       type StatsResult struct {
           TotalSwitches int            `json:"total_switches"`
           Profiles      []ProfileStats `json:"profiles"`
       }
       ```

    2. `AggregateStats(configDir, sinceDate, profileFilter string) (*StatsResult, error)`:
       - Call `ReadUsage(configDir)` to get all entries
       - If `sinceDate` != "": filter entries where `entry.Timestamp >= sinceDate+"T00:00:00"` (lexicographic comparison)
       - If `profileFilter` != "": filter entries where `entry.Profile == profileFilter`
       - Aggregate per profile: count total switches (action == "switch"), track all action counts, record last timestamp
       - Sort profiles by switches descending, then by name alphabetically for ties
       - Return StatsResult with TotalSwitches = sum of all switch counts

    3. `FormatStats(result *StatsResult, sinceLabel string) string`:
       - Match the expected output format:
         ```
         Profile usage (all time):
           work        30 switches  (last: 2h ago)
           personal     8 switches  (last: 3d ago)
         Total: 42 switches across 3 profiles
         ```
       - Use `sinceLabel` instead of "all time" if --since was provided
       - Implement relative time: <60min "Nm ago", <24h "Nh ago", <7d "Nd ago", else "Nw ago"
       - Right-align switch counts for readability

    **Create internal/profile/unmanage.go:**

    1. `UnmanageActive(claudeDir, configPath string, dryRun bool) ([]string, error)`:
       - Read all entries in claudeDir with `os.ReadDir`
       - For each entry: check if it is a symlink with `os.Lstat`
       - Skip non-symlinks (already real files) and protected paths (use `fs.IsProtected`)
       - For symlinks: resolve target with `filepath.EvalSymlinks` (follows all levels)
         - If target is a file: remove symlink, copyFile from target to claudeDir/name
         - If target is a directory: remove symlink, copyDirRecursive from target to claudeDir/name
         - Note: import copyFile from shared.go — it is unexported so UnmanageActive must be in the profile package. Use the existing copyFile and add copyDirRecursive (from Plan 02) or implement inline.
       - After materializing all symlinks: `config.SaveConfig(configPath, config.Config{Active: ""})` to clear active
       - If dryRun: collect list of symlinks that would be materialized, do not modify anything
       - Return list of materialized paths

    **Tests (usage_test.go additions):**
    - TestAggregateStats_CountsSwitches: 5 entries (3 switches for A, 2 for B), verify counts
    - TestAggregateStats_SinceFilter: entries before and after cutoff date, verify filter works
    - TestAggregateStats_ProfileFilter: entries for A and B, filter to A only
    - TestAggregateStats_SortedByCount: verify descending sort
    - TestAggregateStats_EmptyFile: no entries, verify empty result
    - TestFormatStats_Output: verify human-readable format

    **Tests (unmanage_test.go):**
    - TestUnmanageActive_MaterializesSymlinks: create symlinks in tmp claudeDir, run unmanage, verify real files
    - TestUnmanageActive_SkipsRealFiles: regular file in claudeDir stays untouched
    - TestUnmanageActive_SkipsProtected: protected paths (credentials.json) not touched
    - TestUnmanageActive_ClearsConfig: verify config.json has active="" after unmanage
    - TestUnmanageActive_DryRun: verify no filesystem changes
    - TestUnmanageActive_DirectorySymlink: symlink to directory gets materialized as real directory
  </action>

    - Short: "Show profile usage statistics"
    - Args: `cobra.NoArgs`
    - Flags: `--json` (bool), `--since` (string), `--profile` (string)

      1. Resolve configDir with `config.ConfigDir()`
      2. Call `usage.AggregateStats(configDir, since, profileFilter)`

      4. Else: call `usage.FormatStats(result, sinceLabel)`, print

    - Use: `unmanage`
    - Short: "Stop using claudehopper and materialize all symlinks"

    - Args: `cobra.NoArgs`
    - Flags: `--dry-run` (bool)
    - Implementation:
      1. If not dry-run and `isInteractive()`: prompt "This will materialize all symlinks in ~/.claude/ and deactivate claudehopper. Continue? [y/N]"

      3. Call `profile.UnmanageActive(cDir, configPath, dryRun)`

      5. Else: print "Materialized N symlinks. claudehopper is now inactive."
