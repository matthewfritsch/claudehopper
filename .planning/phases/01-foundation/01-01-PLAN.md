  <name>Task 2: Makefile and goreleaser config for dual binary</name>
  <files>Makefile, .goreleaser.yaml</files>
  <action>
    1. Create `Makefile` with the following targets:
       - `GOBIN ?= $(shell go env GOPATH)/bin`
       - `VERSION ?= dev`
       - `COMMIT ?= $(shell git rev-parse --short HEAD 2>/dev/null || echo "none")`
       - `DATE ?= $(shell date -u +%Y-%m-%dT%H:%M:%SZ)`
       - `LDFLAGS = -s -w -X main.version=$(VERSION) -X main.commit=$(COMMIT) -X main.date=$(DATE)`
       - `.PHONY: build` — builds `bin/claudehopper` with ldflags, then `ln -sf claudehopper bin/hop`
       - `.PHONY: install` — `go install -ldflags="$(LDFLAGS)" .` then `ln -sf $(GOBIN)/claudehopper $(GOBIN)/hop`
       - `.PHONY: test` — `go test -v -race ./...`
       - `.PHONY: clean` — removes `bin/`
    2. Create `.goreleaser.yaml` with:
       - Two build stanzas (id: claudehopper and id: hop), both with `main: .`
       - binary names: `claudehopper` and `hop` respectively
       - Same ldflags: `-s -w -X main.version={{.Version}} -X main.commit={{.ShortCommit}} -X main.date={{.Date}}`
       - goos: [linux, darwin] (no windows per renameio limitation)
       - goarch: [amd64, arm64]
       - archives section combining both builds
    3. Verify: `make build` produces both binaries, `bin/claudehopper --version` works, `bin/hop --version` works.
  </action>

  </done>
