  <name>Task 1: Implement tree and diff business logic in internal/profile/</name>
  <files>internal/profile/tree.go, internal/profile/tree_test.go, internal/profile/diff.go, internal/profile/diff_test.go</files>
  <behavior>
    - BuildTree returns a slice of TreeNode with children populated from created_from fields
    - TreeNode includes name, active flag, description, managed count, shared count, shared_paths map, children slice
    - RenderTree produces ASCII output with box-drawing connectors matching the expected format
    - Active profiles annotated with " (active)"
    - Shared files annotated with " (shared from SOURCE)"
    - Cycle detection: visited set prevents infinite recursion
    - TreeJSON returns the rich JSON schema from RESEARCH.md
    - DiffProfiles returns DiffResult with only_a, only_b, common_identical, common_different sets
    - DiffProfiles compares file content for common paths (byte comparison)
    - DiffProfiles handles directories in managed_paths (compare by existence only)
  </behavior>
  <action>
    **internal/profile/tree.go:**

    1. Define `TreeNode` struct:
       ```go
       type TreeNode struct {
           Name         string
           Active       bool
           Description  string
           ManagedCount int
           SharedCount  int
           SharedPaths  map[string]string
           Children     []*TreeNode
       }
       ```

    2. `BuildTree(profilesDir, configPath string) ([]*TreeNode, error)`:
       - Load config for active profile name
       - List all profile dirs with `os.ReadDir(profilesDir)`
       - For each: load manifest, create TreeNode with counts
       - Build `children_of` map from `created_from` fields: `map[string][]*TreeNode`
       - Root nodes = profiles whose created_from is empty or references a non-existent profile
       - Wire children into parent nodes
       - Sort children alphabetically within each parent
       - Return root nodes

    3. `RenderTree(roots []*TreeNode) string`:
       - Recursive rendering using box-drawing connectors
       - For each root: render with `renderNode(node, prefix, isLast, visited, &buf)`
       - Use `visited map[string]bool` for cycle detection — skip if already visited
       - Profile line: `{connector}{name} (N managed, M shared){" (active)" if active}`
       - For each shared path in profile: indent with child prefix + `{connector}{path} (shared from {source})`
       - Return complete string

    4. `TreeJSON(roots []*TreeNode, activeName string) ([]byte, error)`:
       - Build JSON structure matching the schema in RESEARCH.md:
         ```json
         {"active": "...", "profiles": [{name, active, description, created_from, managed_paths, managed_count, shared_paths, shared_count, children}]}
         ```
       - Use `json.MarshalIndent` with 2-space indent

    **internal/profile/diff.go:**

    1. Define `DiffResult` struct:
       ```go
       type DiffResult struct {
           OnlyA     []string // paths only in profile A
           OnlyB     []string // paths only in profile B
           Identical []string // common paths with identical content
           Different []string // common paths with different content
       }
       ```

    2. `DiffProfiles(profilesDir, nameA, nameB string) (*DiffResult, error)`:
       - Load manifests for both profiles
       - Build sets from ManagedPaths
       - Compute OnlyA (in A but not B), OnlyB (in B but not A), Common (in both)
       - For common paths: compare files byte-by-byte
         - Both are regular files: read and compare bytes
         - One or both missing on disk: mark as "different"
         - Both are directories: compare by walking and checking file names
       - Sort all result slices alphabetically

    3. `FormatDiff(result *DiffResult, nameA, nameB string) string`:
       - Match the expected output format:
         ```
         Only in 'nameA':
           file1
         Only in 'nameB':
           file2
         Common:
           file3  [identical]
           file4  [different]
         ```
       - Skip sections that are empty

    - TestBuildTree_SingleRoot: one profile, no created_from
    - TestBuildTree_ParentChild: profile B created_from A
    - TestBuildTree_MultipleRoots: two independent profiles
    - TestBuildTree_Cycle: A created_from B, B created_from A — no infinite loop
    - TestRenderTree: verify ASCII output format with connectors
    - TestRenderTree_ActiveMarker: active profile shows (active)
    - TestRenderTree_SharedAnnotation: shared files show source
    - TestTreeJSON: verify JSON schema structure

    **Tests (diff_test.go):**

    - TestDiffProfiles_Identical: same file content in both
    - TestDiffProfiles_Different: different content in both
    - TestDiffProfiles_Mixed: combination of all categories
    - TestFormatDiff: verify output format string

    Create three Cobra command files following the established pattern (global var + init() + rootCmd.AddCommand).

    **cmd/tree.go — `hop tree [--json]`:**
    - Use: `tree`
    - Short: "Show profile lineage tree"
    - Args: `cobra.NoArgs`
    - Flags: `--json` (bool)
    - Implementation:
      1. Resolve profilesDir and configPath
      2. Call `profile.BuildTree(profilesDir, configPath)`
      3. If `--json`: call `profile.TreeJSON(roots, activeName)`, print JSON
      4. Else: call `profile.RenderTree(roots)`, print formatted output

    **cmd/diff.go — `hop diff <profile-a> <profile-b>`:**
    - Use: `diff PROFILE_A PROFILE_B`
    - Short: "Compare two profiles"
    - Args: `cobra.ExactArgs(2)`
    - Implementation:
      1. Normalize both names with `profile.NormalizeProfileName`
      2. Resolve profilesDir
      3. Call `profile.DiffProfiles(profilesDir, nameA, nameB)`
      4. Print with `profile.FormatDiff(result, nameA, nameB)`

    **cmd/path.go — `hop path <name>`:**
    - Use: `path NAME`
    - Short: "Print profile directory path"
    - Args: `cobra.ExactArgs(1)`
    - Implementation:
      1. Normalize name
      2. Call `config.ProfileDir(name)` to get path
      3. Verify directory exists with `os.Stat` — error if not
      4. `fmt.Println(path)` — just the path, no decoration, for scripting use
