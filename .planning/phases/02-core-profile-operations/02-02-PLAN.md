  <name>Task 1: List profiles and profile status with link health</name>
  <files>internal/profile/list.go, internal/profile/list_test.go, internal/profile/status.go, internal/profile/status_test.go</files>
  <behavior>
    - Test: ListProfiles returns empty slice for empty profiles dir
    - Test: ListProfiles returns sorted ProfileSummary with name, description, managed count, shared count, isActive flag
    - Test: ListProfiles marks the active profile based on config.Active
    - Test: ListProfiles skips directories without a manifest (non-profile dirs)
    - Test: GetProfileStatus returns per-path health: linked (symlink exists and points to profile), shared (symlink to shared dir), conflict (real file exists at link location), not-linked (nothing at link location)
    - Test: GetProfileStatus reports broken symlinks as "broken"
  </behavior>
  <action>
1. Create `internal/profile/list.go`:
   - `type ProfileSummary struct { Name, Description string; ManagedCount, SharedCount int; IsActive bool }`
   - `func ListProfiles(profilesDir, configPath string) ([]ProfileSummary, error)`:
     - Read profilesDir with os.ReadDir
     - For each directory entry, try LoadManifest from `dir/.hop-manifest.json`
     - Skip entries where manifest load fails (not a profile)
     - Load config to check active profile name
     - Return sorted slice of ProfileSummary
   - `func FormatProfileList(summaries []ProfileSummary) string` - format matching Python: `  name (active)  [N paths, M shared] - desc`

2. Create `internal/profile/status.go`:
   - `type PathHealth struct { Name, Status, Detail string }` where Status is one of: "linked", "shared", "conflict", "not-linked", "broken"
   - `type ProfileStatusInfo struct { Name, Description string; Paths []PathHealth }`
   - `func GetProfileStatus(profileDir, claudeDir, sharedDir string, manifest config.Manifest) ProfileStatusInfo`:
     - For each managed_path in manifest: os.Lstat the expected symlink location in claudeDir
     - If symlink: readlink and check if target is in profileDir ("linked") or sharedDir ("shared") or dangling ("broken")
     - If regular file/dir exists: "conflict"
     - If nothing exists: "not-linked"
     - For shared_paths: check symlink health similarly
   - `func FormatProfileStatus(info ProfileStatusInfo) string` - format matching Python: per-path `[linked]`, `[linked, shared from X]`, `[CONFLICT]`, `[not linked]`

3. All functions accept explicit directory paths (testable with t.TempDir()).

4. Create `internal/profile/list_test.go` using t.TempDir() with:
   - Empty profiles dir
   - Multiple profiles with manifests, one active
   - Non-profile directory (no manifest) is skipped

5. Create `internal/profile/status_test.go` using t.TempDir() with:
   - Set up profileDir with files, claudeDir with symlinks pointing to them
   - Test each health state: linked, shared, conflict, not-linked, broken
  </action>
  <verify>
    <automated>cd /home/matthew/Programming/claudehopper-go && go test ./internal/profile/... -run "TestList|TestStatus|TestFormat" -v</automated>
  </verify>
  <done>ListProfiles returns correct summaries for profiles dir. GetProfileStatus correctly identifies all link health states. Format functions produce Python-matching output.</done>
</task>
